JavaScript const Keyword (ES6 / 2015)

1. Introduction

The const keyword was introduced in ES6 (2015). Variables defined with const have block scope, cannot be redeclared, and cannot be reassigned. const ensures that the variable reference remains constant.

2. Cannot be Reassigned

Variables declared with const cannot be reassigned:


// Example
const PI = 3.141592653589793;
PI = 3.14;      // ERROR: Cannot reassign
PI = PI + 10;   // ERROR: Cannot reassign

3. Must be Assigned When Declared

Variables must have a value assigned at the time of declaration:


// Correct
const PI = 3.14159265359;

// Incorrect
const PI2;       // ERROR: Missing initializer
PI2 = 3.1415;

4. Block Scope

Variables declared with const exist only within the block they are defined:


{
    const a = 10;
    console.log(a); // 10
}
console.log(a); // ERROR: a is not defined

5. Constant Arrays

You can modify elements of a constant array, but you cannot reassign the array itself:


// Create a constant array
const cars = ["Saab", "Volvo", "BMW"];

// Modify an element
cars[0] = "Toyota";

// Add a new element
cars.push("Audi");

// Reassign the array (not allowed)
cars = ["Toyota", "Volvo", "Audi"]; // ERROR

6. Constant Objects

You can modify properties of a constant object, but you cannot reassign the object itself:


// Create a constant object
const car = {type: "Fiat", model: "500", color: "white"};

// Change a property
car.color = "red";

// Add a new property
car.owner = "Johnson";

// Reassign the object (not allowed)
car = {type: "Volvo", model: "EX60", color: "red"}; // ERROR

7. Redeclare in Same Scope

You cannot redeclare a variable with const in the same block:


const name = "Alice";
const name = "Bob"; // ERROR: Identifier 'name' has already been declared

8. Hoisting and Temporal Dead Zone

Const is hoisted but cannot be used before declaration (temporal dead zone):


console.log(myConst); // ERROR: Cannot access 'myConst' before initialization
const myConst = 100;

9. When to Use const

Use const when the variable value should not be reassigned:


// Array
const numbers = [1, 2, 3];

// Object
const person = {name: "John", age: 30};

// Function
const greet = function() {
    console.log("Hello");
};

// RegExp
const pattern = /abc/;

10. Extra Notes / Best Practices

  • Const does not make objects immutable: Object properties or array elements can still be changed. 
    
const arr = [1, 2, 3];
arr[0] = 10;  // Allowed
arr.push(4);  // Allowed
arr = [5,6];  // ERROR
  • Use Object.freeze() const obj = Object.freeze({name:"John"}); obj.name = "Alice"; // Ignored, object is frozen
  • Always prefer const for values that should not change, and let for variables that will change.
